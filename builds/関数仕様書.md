## 1. 概要

**IpatHelper** は、JRA（日本中央競馬会）のインターネット投票サービス「I-PAT」および地方競馬の投票システムに対して、自動ログイン・入出金・馬券購入・購入履歴取得などを行うための C++ DLL ライブラリです。

- Windows DLL として提供。C 言語インターフェースにより他言語からも呼び出し可能。
- 内部では libcurl を使用して HTTPS 通信を行います。
- 中央競馬（I-PAT）と地方競馬の両システムに対応しています。

## 2. 共通仕様

### 2.1 戻り値

全関数（`void` 型を除く）は `unsigned int` をビットフラグとして返します。複数のフラグが同時に立つ場合があります。判定には AND 演算を使用してください。

| 定数名 | 値 (bit) | 説明 |
|---|---|---|
| `SUCCESS` | `0b00000001` | 処理に成功 |
| `UNSUCCESS` | `0b00000010` | 処理に失敗 |
| `FAILED_CHUOU` | `0b00000100` | 中央競馬での処理に失敗 |
| `FAILED_CHIHOU` | `0b00001000` | 地方競馬での処理に失敗 |
| `FAILED_COMMUNICATE_CHUOU` | `0b00010000` | 中央競馬での通信失敗（IPAT レスポンスエラー） |
| `FAILED_COMMUNICATE_CHIHOU` | `0b00100000` | 地方競馬での通信失敗（IPAT レスポンスエラー） |

**判定例:**

```cpp
if (ret & (unsigned int)RETURN_VALUE::SUCCESS)      { /* 成功 */ }
if (ret & (unsigned int)RETURN_VALUE::FAILED_CHUOU) { /* 中央競馬で失敗 */ }
```

### 2.2 スレッドセーフ

全関数は内部でクリティカルセクションにより排他制御されています。複数スレッドからの同時呼び出しは安全ですが、関数はブロッキング動作となります。

### 2.3 前提条件

馬券購入・入出金・履歴取得の各関数は、事前に `Login()` が成功している必要があります。未ログイン状態での呼び出しは `UNSUCCESS` を返します。

## 5. 関数仕様

### 5.1 Login — ログイン

```cpp
unsigned int Login(
    const char szINetId[],
    const char szId[],
    const char szPassword[],
    const char szPars[]
);
```

I-PAT（中央競馬）および地方競馬の両システムへ並行してログインします。一方が成功した時点で `SUCCESS` フラグが立ちます。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `szINetId` | `const char[]` | I-NET ID（JRA 発行のインターネット ID） |
| `szId` | `const char[]` | ログイン ID（加入者番号） |
| `szPassword` | `const char[]` | パスワード |
| `szPars` | `const char[]` | P-ARS 番号（暗証番号） |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 両方成功 | `SUCCESS` |
| 中央のみ失敗 | `SUCCESS \| FAILED_CHUOU` |
| 地方のみ失敗 | `SUCCESS \| FAILED_CHIHOU` |
| 両方失敗 | `FAILED_CHUOU \| FAILED_CHIHOU` |
| 既ログイン | `UNSUCCESS` |

> **注意:** 既にログイン状態の場合は `UNSUCCESS` を返します（二重ログイン不可）。

---

### 5.2 Logout — ログアウト

```cpp
unsigned int Logout();
```

I-PAT および地方競馬の両システムからログアウトし、通信ハンドルやセッション情報をリセットします。自動入金設定もリセットされます。

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功 | `SUCCESS` |
| 未ログイン | `UNSUCCESS` |

> **注意:** `Logout()` 後に再度 `Login()` すれば再利用可能です。

---

### 5.3 Deposit — 入金

```cpp
unsigned int Deposit(
    const unsigned int unDepositValue
);
```

登録口座から I-PAT 残高へ入金します。中央競馬へのログインがある場合は中央系から試行し、失敗時は地方系へフォールバックします。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `unDepositValue` | `const unsigned int` | 入金額（円）。100 円以上かつ 100 円単位。 |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功 | `SUCCESS` |
| 未ログイン | `UNSUCCESS` |
| 金額不正（100円未満 / 100円単位でない） | `UNSUCCESS` |
| 中央失敗・地方成功 | `FAILED_CHUOU \| SUCCESS` |
| 両方失敗 | `FAILED_CHUOU \| FAILED_CHIHOU` |

> **注意:** 金額は **100 円以上かつ 100 円単位** で指定してください。

---

### 5.4 Withdraw — 出金

```cpp
unsigned int Withdraw();
```

I-PAT 残高を登録口座へ全額出金します。中央ログイン時は中央系から試行し、失敗時は地方系へフォールバックします。

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功 | `SUCCESS` |
| 未ログイン | `UNSUCCESS` |
| 中央失敗・地方成功 | `FAILED_CHUOU \| SUCCESS` |
| 両方失敗 | `FAILED_CHUOU \| FAILED_CHIHOU` |

> **注意:** 出金額の指定は不要です。残高の全額が対象となります。

---

### 5.5 GetPurchaseData — 馬券購入履歴取得

```cpp
unsigned int GetPurchaseData(
    ST_PURCHASE_DATA* pobjStatus
);
```

現在のセッションにおける馬券購入履歴（残高・累計金額・購入済み馬券一覧）を取得します。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `pobjStatus` | `ST_PURCHASE_DATA*` | **【出力】** 馬券購入履歴を格納する構造体へのポインタ。呼び出し前に確保しておくこと。 |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功（`pobjStatus` に結果が設定される） | `SUCCESS` |
| `pobjStatus` が `nullptr` | `UNSUCCESS` |
| 未ログイン | `UNSUCCESS` |
| 通信失敗 | `FAILED_CHUOU` / `FAILED_CHIHOU` など |

> **注意:** 取得した構造体は必ず **`ReleasePurchaseData()` で解放**してください。

---

### 5.6 ReleasePurchaseData — 馬券購入履歴解放

```cpp
void ReleasePurchaseData(
    ST_PURCHASE_DATA* pobjStatus
);
```

`GetPurchaseData()` が内部で確保したメモリを解放します。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `pobjStatus` | `ST_PURCHASE_DATA*` | `GetPurchaseData()` に渡した構造体へのポインタ |

> **注意:**
> - `GetPurchaseData()` の成否にかかわらず**必ず呼び出してください**。
> - `pobjStatus` が `nullptr` の場合は何もしません。
> - 内部ポインタは解放後に `nullptr` にセットされます（二重解放防止）。

### 5.7 GetBetInstance — 馬券購入情報の構築

```cpp
unsigned int GetBetInstance(
    const unsigned short usPlace,
    const unsigned char  ucRaceNo,
    const unsigned short usYear,
    const unsigned char  ucMonth,
    const unsigned char  ucDay,
    const unsigned char  ucHoushiki,
    const unsigned char  ucShikibetsu,
    const unsigned int   unKingaku,
    const char           szKaime[],
    ST_BET_DATA*         pobjBetData
);
```

買い目文字列を解析し、`ST_BET_DATA` 構造体を構築します。構築した構造体を `Bet()` に渡すことで馬券を購入します。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `usPlace` | `const unsigned short` | 開催場（`KAISAI` 列挙値） |
| `ucRaceNo` | `const unsigned char` | レース番号 |
| `usYear` | `const unsigned short` | 開催年（西暦 4 桁） |
| `ucMonth` | `const unsigned char` | 開催月（1〜12） |
| `ucDay` | `const unsigned char` | 開催日（1〜31） |
| `ucHoushiki` | `const unsigned char` | 方式（`HOUSHIKI` 列挙値） |
| `ucShikibetsu` | `const unsigned char` | 式別（`SHIKIBETSU` 列挙値） |
| `unKingaku` | `const unsigned int` | 1 点あたりの購入金額（円）。100 円単位。 |
| `szKaime` | `const char[]` | 買い目文字列（書式は下記参照） |
| `pobjBetData` | `ST_BET_DATA*` | **【出力】** 購入情報を格納する構造体へのポインタ |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功（`pobjBetData` に購入情報が設定される） | `SUCCESS` |
| `pobjBetData` または `szKaime` が `nullptr` / 不正値 | `UNSUCCESS` |

**買い目文字列（`szKaime`）の書式:**

- 列（軸）の区切り: **`-`**（ハイフン）
- 同一列内の複数馬番: **`,`**（カンマ）

| 式別 / 方式 | 例 | 意味 |
|---|---|---|
| 単勝・複勝 | `"1,2,3"` | 1 番・2 番・3 番 |
| 馬連フォーメーション | `"1,2-3,4"` | 軸 1・2 番 vs 3・4 番 |
| 三連単フォーメーション | `"1-2,3-4,5"` | 1 着: 1番 / 2 着: 2・3番 / 3 着: 4・5番 |
| ボックス | `"1,2,3"` | 1・2・3 番のボックス |

### 5.8 Bet — 馬券購入

```cpp
unsigned int Bet(
    const ST_BET_DATA    pobjBetData[],
    const unsigned short usBetCount,
    const unsigned short usWaitMilliSeconds = 500
);
```

`GetBetInstance()` で構築した `ST_BET_DATA` の配列を一括購入します。開催場に応じて自動的に中央・地方・海外に振り分けます。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `pobjBetData[]` | `const ST_BET_DATA[]` | 購入情報の配列（`GetBetInstance` で構築したもの） |
| `usBetCount` | `const unsigned short` | 配列の要素数 |
| `usWaitMilliSeconds` | `const unsigned short` | 購入間隔（ms）。デフォルト: **500ms** |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 全件成功 | `SUCCESS` |
| 中央失敗 | `FAILED_CHUOU` |
| 地方失敗 | `FAILED_CHIHOU` |
| 未ログイン / パラメータ不正 | `UNSUCCESS` |

> **注意:** 購入間隔（`usWaitMilliSeconds`）が短すぎると購入に失敗することがあります。ネットワーク状況に応じてデフォルト値 500ms を基準に調整してください。

### 5.10 BetWin5 — WIN5 馬券購入

```cpp
unsigned int BetWin5(
    const ST_BET_DATA_WIN5 objBetData,
    const unsigned short   usWaitMilliSeconds = 500
);
```

`GetBetInstanceWin5()` で構築した `ST_BET_DATA_WIN5` を使って WIN5 を購入します。組合せ数が 1 回あたりの上限を超える場合は内部で自動分割して購入します。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `objBetData` | `const ST_BET_DATA_WIN5` | WIN5 購入情報（`GetBetInstanceWin5` で構築したもの） |
| `usWaitMilliSeconds` | `const unsigned short` | 購入間隔（ms）。デフォルト: **500ms** |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功 | `SUCCESS` |
| 中央競馬に未ログイン | `UNSUCCESS` |
| 通信失敗 | `FAILED_COMMUNICATE_CHUOU` |

> **注意:** WIN5 は**中央競馬システムでのみ購入可能**です。地方競馬のみにログインしている場合は `UNSUCCESS` を返します。

### 5.11 SetAutoDepositFlag — 自動入金設定

```cpp
unsigned int SetAutoDepositFlag(
    const bool           bEnable,
    const unsigned int   unDepositValue  = 1000,
    const unsigned short usConfirmTimeout = 10000
);
```

馬券購入前の自動入金機能を設定します。有効化すると、残高が不足している場合に自動的に指定金額を入金してから購入を試みます。入金完了の確認は `usConfirmTimeout` ミリ秒後に行われます。

**パラメータ:**

| パラメータ名 | 型 | 説明 |
|---|---|---|
| `bEnable` | `const bool` | `true`: 自動入金を有効化 / `false`: 無効化 |
| `unDepositValue` | `const unsigned int` | 自動入金額（円）。デフォルト: **1000 円** |
| `usConfirmTimeout` | `const unsigned short` | 入金確認タイムアウト（ms）。デフォルト: **10000ms** |

**戻り値:**

| 状態 | 返るフラグ |
|---|---|
| 成功 | `SUCCESS` |
| 金額不正（100円未満 / 100円単位でない） | `UNSUCCESS` |

> **注意:** `Logout()` を呼び出すと設定はリセットされます。

## 8. エラーハンドリングガイド

戻り値は複数のビットフラグが同時に立つ場合があります。AND 演算で各フラグを個別に確認してください。

```cpp
unsigned int ret = Bet(betData, count);

bool ok       = (ret & (unsigned int)RETURN_VALUE::SUCCESS)                != 0;
bool chuouNG  = (ret & (unsigned int)RETURN_VALUE::FAILED_CHUOU)           != 0;
bool chihouNG = (ret & (unsigned int)RETURN_VALUE::FAILED_CHIHOU)          != 0;
bool netNGC   = (ret & (unsigned int)RETURN_VALUE::FAILED_COMMUNICATE_CHUOU) != 0;
```

| 状況 | 原因 | 対処方法 |
|---|---|---|
| `UNSUCCESS` が立つ | パラメータ不正 or 未ログイン | 引数を確認し、`Login()` が成功しているかチェックする |
| `FAILED_CHUOU` が立つ | 中央競馬系の処理失敗 | `SUCCESS` フラグも確認する。地方系は成功している可能性がある |
| `FAILED_CHIHOU` が立つ | 地方競馬系の処理失敗 | `SUCCESS` / `FAILED_CHUOU` フラグも確認する |
| `FAILED_COMMUNICATE_CHUOU` が立つ | ネットワーク障害 or IPAT サーバーエラー | 時間をおいて再試行。または `Logout()` → `Login()` で再接続する |
| `FAILED_COMMUNICATE_CHIHOU` が立つ | ネットワーク障害 or 地方サーバーエラー | 時間をおいて再試行。または `Logout()` → `Login()` で再接続する |
